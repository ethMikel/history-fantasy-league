# AI 활용 프롬프트 아카이브 — 히스토리 판타지 리그

제출물 ④ "AI 활용 기술 문서"의 1차 소재. 개발 전 과정에서 AI(Claude Code)에 던진 주요 프롬프트·워크플로우·지시를 시간순으로 적재한다. **마지막 날 급조 방지 — 작업하며 계속 기록.**

## 개발 방법론 개요 (비개발자 1인 × AI 디렉팅)
게임 개발·코딩 경험이 없는 1인이 Claude Code를 디렉팅해 게임을 설계·구현. 핵심 워크플로우:
1. **`/grill`** — 큰 작업 전 적대적 질문으로 설계 의도 정렬 (Frederick Brooks "보이지 않는 공유 모델")
2. **병렬 리서치 워크플로우** — 게임개발·산식·레퍼런스·데이터·내러티브를 서브에이전트 팬아웃으로 조사 (아래 목록)
3. **spec-first** — 06_SIM_SPEC(수치 계약서)를 먼저 확정, 코드는 그 구현체
4. **모델 배분** — 판단·설계·재미판정=상위 모델 / 구현 볼륨=효율 모델

## AI 콘텐츠 파이프라인 (Phase 2) — ④ 문서 핵심
실존 인물 데이터를 "AI 생성"이 아니라 "AI 수집·검증·정량화"로 구축:
1. **수집**: MIT Pantheon 데이터셋(126,583명) → 문명×시대 셀 매핑으로 후보 156명 선정
2. **보강**: Wikidata SPARQL/API + 한국어 위키 REST로 한국어명·초상·사료 요약 조인
3. **스코어링**: 6축 앵커 루브릭(고정점 인물) + 근거 인용 강제 + 시대맥락 평정을 12배치 병렬 서브에이전트로
4. **초상**: 퍼블릭 도메인 원본 → Pillow 저해상+팔레트 감색+Floyd-Steinberg 디더로 픽셀 톤 통일

### 스코어링 프롬프트 (핵심 지시 — hfl-scoring 워크플로우)
> 6축(무력·지략·내정·외교·과학·문화)을 0-99로. 5밴드(A 92-99 인류사 정점 ~ E 5-37 무관). 고정 앵커에 맞춰 상대 평정(알렉산드로스 무력98, 뉴턴 과학99, 세종 내정96…). 철칙: ①시대맥락 내 평정(고대·현대 절대 비교 금지) ②축별 독립 근거 강제(유명해도 근거 없는 축은 40 이하 — halo 차단) ③유명세는 능력이 아니다, 스탯 반영 금지.

## 개발 규율 (repo CLAUDE.md)
게임 로직 Math.random 금지(시드 PRNG), 밸런스 상수 단일 파일, 시뮬 순수함수, 애니메이션 transform/opacity만, 라우터 없음, 에셋=CREDITS 동시 기록 등 10개 불가침 규칙.
